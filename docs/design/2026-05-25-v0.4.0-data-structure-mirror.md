# v0.4.0 — Full Data Structure Mirror (shioaji ↔ binance) — Design

**Date**: 2026-05-25
**Status**: Draft (pending design review)
**Target release**: `binance-shioaji-sdk` v0.4.0 (breaking)
**Predecessor**: v0.3.0 (class rename `BinanceClient → Binance`, complete method name mirror)
**Driver**: lcz-quant-core PR2 design surfaced that "mirror shioaji shape" was only **method-name deep** — return types diverge (dict vs Pydantic object; combined vs decomposed). Adapter code for downstream consumers becomes asymmetric.

---

## 1. Context & motivation

### What was claimed (v0.3.0 release notes / README)

> "Async Python SDK for Binance USDM-M Futures, **mirroring the shioaji SDK API shape**. If you already trade 台股期貨 via shioaji, this gives you Binance crypto with the **exact same API surface** — same Contracts.Perp, same Order(action=…), same place_order. Switch contracts, keep your strategy code."

### What's actually mirrored vs not

| Aspect | Mirrored? | Evidence |
|---|---|---|
| Class name (`Binance` ↔ `Shioaji`) | ✅ since v0.3.0 | `from binance_shioaji_sdk import Binance` |
| Method names (`place_order / cancel_order / login / account_balance / list_positions / list_trades`) | ✅ | direct alignment with `sj.*` |
| Method arg signatures | ✅ mostly | `place_order(contract, order)` etc |
| `Contracts.Perp` namespace (vs shioaji's `Contracts.Futures`) | ✅ analogous | accessor pattern matches |
| **Return value types** | ❌ NOT mirrored | shioaji returns Pydantic objects; binance-shioaji-sdk returns dicts |
| **Field decomposition** (which model holds what) | ❌ NOT mirrored | shioaji splits `AccountBalance` + `Margin`; binance-shioaji-sdk merges into single dict |
| **Field names** | ❌ NOT mirrored | shioaji `code/price/pnl`; binance-shioaji-sdk `symbol/avg_price/unrealized_pnl` |
| **Status enum values** | ❌ NOT mirrored | shioaji `Submitted/PartFilled/Filled/Cancelled/Failed`; binance-shioaji-sdk `submitted/partially_filled/filled/cancelled/rejected` |

**Verdict**: mirror is shallow. Consumers (lcz-quant-core BrokerAdapter layer being the first real one) cannot write symmetric code over both vendors; each adapter must know vendor-specific field shapes.

### Why fix it now

lcz-quant-core PR2 brainstorm hit this directly when writing `ShioajiAdapter` + `BinanceAdapter`. Three rounds of plan-review caught real bugs (wrong dict keys; `account_balance()` doesn't have `available_margin` on shioaji side; etc.). Symmetric adapter code is impossible until SDK returns mirror-aligned types.

The "complete" mirror lets adapter code look like:
```python
async def get_balance(self) -> Balance:
    ab = await maybe_await(self._session.api.account_balance)         # both: AccountBalance object
    mg = await maybe_await(self._session.api.margin, account)         # both: Margin object
    return Balance(
        currency=...,
        equity=Decimal(str(ab.acc_balance)),
        available_margin=Decimal(str(mg.available_margin)),
        initial_margin=Decimal(str(mg.initial_margin)),
        maintenance_margin=Decimal(str(mg.maintenance_margin)),
    )
```
Identical body across `ShioajiAdapter` and `BinanceAdapter` — only the `maybe_await` glue + `currency` constant differ.

## 2. Scope

### In scope (v0.4.0, breaking)

1. **New dataclasses** mirroring shioaji return models:
   - `BinanceAccountBalance` — mirror of `sj.AccountBalance` (essential fields only)
   - `BinanceMargin` — mirror of `sj.Margin` (essential margin/equity fields; not all 25 obscure ones)
   - `BinanceFuturePosition` — mirror of `sj.FuturePosition`
   - `BinanceTrade` — mirror of `sj.Trade` (composite: contract + order + status)
   - `BinanceTradeStatus` — mirror of `sj.Trade.status` substructure
   - `BinanceOrderStatusEnum` — mirror of `sj.OrderStatus` enum (CamelCase values: `Submitted / PartFilled / Filled / Cancelled / Failed`)
2. **Return-type changes** (breaking):
   - `account_balance()` → `BinanceAccountBalance` (was `dict`)
   - `list_positions(account)` → `list[BinanceFuturePosition]` (was `list[dict]`)
   - `place_order(contract, order)` → `BinanceTrade` (was `OrderResponse`)
3. **NEW method**: `margin(account)` → `BinanceMargin` (currently missing; required to mirror shioaji's `sj.margin(account)`)
4. **Field-name alignment** (within new dataclasses): use shioaji vocabulary — `code` not `symbol`, `price` not `avg_price`, `pnl` not `unrealized_pnl`, `acc_balance` not `equity` in AccountBalance
5. **Enum alignment**: `BinanceOrderStatusEnum.Submitted` (CamelCase to match `sj.OrderStatus.Submitted`), not `"submitted"` lowercase string
6. **Existing `OrderResponse` deprecated** — emits `DeprecationWarning` for one release, removed v0.5.0. Callers should migrate to `BinanceTrade`.
7. **README + examples** updated to reflect new return types
8. **Version bump** 0.3.0 → 0.4.0; tag + GitHub release

### Out of scope

- shioaji Margin has 25+ fields (collateral_amount, deposit_withdrawal, fee, future_open_position, future_settle_profitloss, option_*, royalty_revenue_expenditure, security_collateral_amount, today_*, yesterday_*, etc.). Mirror **essential subset only** (~6 fields). Binance API doesn't have direct equivalents for most; we don't fake them.
- shioaji Stock-specific models (`StockPosition`, `StockProfitLoss`, etc.) — Binance is futures-only, no stock equivalent needed.
- shioaji `sj.Order` field-by-field mirror — current binance Order dataclass (`price/quantity/action/price_type/...`) is fine; rename is optional.
- WebSocket / Quote subsystem changes — out of scope (v0.4.0 is REST data-shape only).
- Backwards-compat shim that returns dict on old method while object on new — too messy; clean break with one deprecation cycle.

## 3. Side-by-side comparison: shioaji ↔ binance-shioaji-sdk v0.4.0

### 3.1 AccountBalance

| Field | `sj.AccountBalance` (live) | `BinanceAccountBalance` (v0.4.0) | Binance source |
|---|---|---|---|
| `acc_balance` | float | float | `totalWalletBalance` from `/fapi/v2/balance` |
| `date` | str | str | today's date (set by SDK) |
| `errmsg` | str | str (default `""`) | "" on success; error message on failure |
| `status` | str | str | "200" on success; HTTP status code on failure |

**Decomposition**: equity + margin info NOT here (matches shioaji); see `BinanceMargin`.

### 3.2 Margin (NEW for binance)

| Field | `sj.Margin` (live, partial — 25 total) | `BinanceMargin` (v0.4.0) | Binance source |
|---|---|---|---|
| `available_margin` | float | float | `availableBalance` from `/fapi/v2/account` |
| `initial_margin` | float | float | `totalInitialMargin` |
| `maintenance_margin` | float | float | `totalMaintMargin` |
| `equity` | float | float | `totalMarginBalance` (equity = wallet + uPnL) |
| `equity_amount` | float | float | alias of equity (shioaji keeps both; we keep for symmetry) |
| `today_balance` | float | float (=acc_balance from /balance call) | redundant with AccountBalance; kept for shioaji parity |
| `yesterday_balance` | float | float (best-effort; Binance doesn't track daily snapshot) | optional / 0.0 if unavailable |
| `status` | str | str | "200" on success |

**Out-of-scope shioaji-only fields**: `collateral_amount / deposit_withdrawal / fee / option_*` etc. — Binance has no equivalent; would be lying to mirror.

### 3.3 FuturePosition

| Field | `sj.FuturePosition` (live) | `BinanceFuturePosition` (v0.4.0) | Binance source |
|---|---|---|---|
| `code` | str | str | `symbol` from `/fapi/v2/positionRisk` (rename `symbol → code` per shioaji vocab) |
| `direction` | str (`"Buy"` / `"Sell"`) | str (`"Buy"` / `"Sell"`) | derived from sign of `positionAmt` (rename `"long"/"short"` → `"Buy"/"Sell"` for shioaji vocab) |
| `id` | str | str | `symbol`-as-id OR position UUID; Binance doesn't have native position ID — synthesize as `f"{symbol}_{positionSide}"` |
| `last_price` | float | float | `markPrice` (Binance reports both markPrice and lastPrice; markPrice is the canonical mark) |
| `pnl` | float | float | `unRealizedProfit` |
| `price` | float | float | `entryPrice` |
| `quantity` | int | int | `abs(int(positionAmt))` (shioaji uses int contracts; Binance allows fractional like 0.001 — see §4 risk) |

**Vocabulary rename**: `direction` was `"long"/"short"` in v0.3.0; aligns to shioaji `"Buy"/"Sell"`.

### 3.4 Trade + TradeStatus + Order (place_order return)

`sj.Trade` is a composite: `trade.contract`, `trade.order`, `trade.status`.

`BinanceTrade` mirrors this structure:

```python
@dataclass(frozen=True)
class BinanceTradeStatus:
    """Mirror of sj.Trade.status — order lifecycle snapshot."""
    status: BinanceOrderStatusEnum      # Submitted / PartFilled / Filled / Cancelled / Failed
    status_code: str                     # broker reason code (Binance: empty on success)
    order_datetime: str                  # ISO timestamp
    deal_quantity: int                   # cumulative filled qty
    modified_price: float                # weighted avg fill price (0.0 if no fill)

@dataclass(frozen=True)
class BinanceTrade:
    """Mirror of sj.Trade. Returned by place_order()."""
    contract: BinanceContract            # the contract that was traded (existing class)
    order: Order                         # the order params submitted (existing class; consider rename?)
    status: BinanceTradeStatus           # current status snapshot
```

Fields rationale:

| sj.Trade.status field | BinanceTradeStatus field | Binance source |
|---|---|---|
| `id` (order id) | (in `order.id` per shioaji; binance puts it on `order`) — see Order below | `orderId` from POST /fapi/v1/order |
| `status` (OrderStatus enum) | `status` (BinanceOrderStatusEnum) | `status` field of order response |
| `status_code` | `status_code` (default `""`) | not directly in Binance response; populated on error |
| `order_datetime` | `order_datetime` | `updateTime` epoch ms → ISO |
| `deal_quantity` | `deal_quantity` | `executedQty` |
| `modified_price` | `modified_price` | `avgPrice` (0.0 if no fill) |

**Order id**: shioaji's `Trade.order.id` is the broker-assigned unique. v0.4.0 puts the Binance order id in `Order.id` post-submit (new `id` field on existing Order dataclass — `Optional[str]`, default None for pre-submit construction).

### 3.5 BinanceOrderStatusEnum

Mirror `sj.OrderStatus` exactly (CamelCase, identical member names):

```python
class BinanceOrderStatusEnum(StrEnum):
    Cancelled = "Cancelled"
    Failed = "Failed"
    Filled = "Filled"
    Inactive = "Inactive"
    PartFilled = "PartFilled"
    PendingSubmit = "PendingSubmit"
    PreSubmitted = "PreSubmitted"
    Submitted = "Submitted"
```

Binance's `status` field returns: `NEW / PARTIALLY_FILLED / FILLED / CANCELED / REJECTED / EXPIRED`. Mapping:

| Binance API status | BinanceOrderStatusEnum |
|---|---|
| `NEW` | `Submitted` |
| `PARTIALLY_FILLED` | `PartFilled` |
| `FILLED` | `Filled` |
| `CANCELED` | `Cancelled` |
| `REJECTED` | `Failed` |
| `EXPIRED` | `Cancelled` (or new `Expired` if we want more granularity; for v0.4.0 collapse into Cancelled) |
| (no equivalent) | `Inactive`, `PendingSubmit`, `PreSubmitted` — present in enum for shioaji parity but never produced by Binance |

### 3.6 list_trades return

`sj.list_trades(...)` returns `list[Trade]`. `binance.list_trades(...)` v0.3.0 returns broker-specific dicts. v0.4.0: `list[BinanceTrade]` (or a new `BinanceFilledTrade` if shioaji has a separate "filled trade history" model — TBD by writing-plans verification).

## 4. Risks

| Risk | Mitigation |
|---|---|
| Binance allows fractional quantity (`0.001 BTC`); shioaji is integer contracts. `BinanceFuturePosition.quantity: int` truncates fractional positions. | Use `Decimal` instead of `int` for quantity in `BinanceFuturePosition`. Diverges from shioaji's int contract — but trying to mirror "int" would silently round 0.5 BTC to 0. Honesty > mirror precision here. Document divergence in field docstring. |
| Binance has no native "position ID" — synthesis `f"{symbol}_{positionSide}"` is fragile across hedge mode. | Default to one-way mode initially; document in v0.4.0 README. Hedge mode → separate ID scheme in v0.5.0. |
| shioaji `Margin.today_balance` ≈ `acc_balance`; redundant duplication in two models. | Keep for shioaji parity — caller may query either; cheaper than re-fetching. |
| `BinanceOrderStatusEnum` has members never produced by Binance (`Inactive / PendingSubmit / PreSubmitted`). | Document as "shioaji-parity only" in docstring. Type system allows them; runtime never returns them from Binance side. |
| `BinanceTrade` composite breaks downstream callers of `OrderResponse`. | Deprecation cycle: v0.4.0 `place_order()` returns `BinanceTrade`; old `OrderResponse` accessible via `trade._legacy_response` property with `DeprecationWarning`. Removed v0.5.0. (Alternative: hard break in v0.4.0 since lcz-quant-core is the only known consumer and we control both repos.) |
| lcz-quant-core PR2 in-flight will need full rewrite of BinanceAdapter to consume new types. | Expected; that's the whole point of this release. PR2 design + plan get re-baselined post-v0.4.0. |
| New `margin(account)` method needs a Binance REST endpoint that returns margin breakdown. | `/fapi/v2/account` returns `totalInitialMargin / totalMaintMargin / availableBalance / totalMarginBalance`. Already used internally — just expose as method. |

## 5. Backwards compatibility

**Decision: clean break + 1-release deprecation for `OrderResponse` only.**

Rationale:
- Only known consumer is lcz-quant-core (PR2 in-flight; we control it; bumps pin in coordinated PR)
- sentinel uses `lcz_binance_sdk` (different package, not affected)
- Maintaining dict-OR-object dual return for `account_balance / list_positions` adds 100+ lines of compat shim for zero real benefit

Migration path (in v0.4.0 release notes):
```python
# Before (v0.3.x):
balance_dict = await api.account_balance()
print(balance_dict["equity"])

# After (v0.4.0):
balance = await api.account_balance()
print(balance.acc_balance)              # mirror shioaji
# (margin info moved to separate call:)
mg = await api.margin(api.futures_account)
print(mg.available_margin)
```

```python
# Before (v0.3.x):
resp = await api.place_order(contract, order)
order_id = resp.order_id

# After (v0.4.0):
trade = await api.place_order(contract, order)
order_id = trade.order.id              # mirror sj.Trade.order.id
status = trade.status.status            # mirror sj.Trade.status.status
```

## 7. Test strategy

### Unit (existing — update mocks to new return shape)
- `test_client.py` — update `place_order` mock to return `BinanceTrade`-shaped data; `account_balance` to return `BinanceAccountBalance`-shaped; etc.
- `test_account.py` — assertions on dataclass attrs, not dict keys.

### New mirror-symmetry tests (the headline of v0.4.0)
- `test_shioaji_mirror.py` — for each (binance dataclass, shioaji model) pair:
  ```python
  def test_binance_account_balance_mirrors_shioaji():
      import shioaji as sj
      from binance_shioaji_sdk import BinanceAccountBalance
      sj_fields = set(a for a in dir(sj.AccountBalance) if not a.startswith('_') and a not in ('dict', 'keys'))
      bn_fields = set(f.name for f in dataclasses.fields(BinanceAccountBalance))
      missing = sj_fields - bn_fields
      assert not missing, f"BinanceAccountBalance missing shioaji fields: {missing}"
  ```
  Same for `BinanceMargin / BinanceFuturePosition / BinanceOrderStatusEnum`.
  
  These tests **import shioaji as a test-only dependency** — `pyproject.toml [project.optional-dependencies]` add `mirror-test = ["shioaji"]`; CI installs the optional group.
  
  Future shioaji releases adding fields → test surfaces drift → maintainer decides whether to mirror.

### Manual / smoke
- One real call against Binance testnet (PAPER mode): `account_balance()` returns valid `BinanceAccountBalance`; `margin(account)` returns valid `BinanceMargin`. (Existing manual smoke pattern.)

## 8. Open decisions

- **`Order.id` field added** (`Optional[str], default None`) to carry order id post-submit, vs. require accessing via `trade.order.id`. Per design §3.4 — choose adding `id` to existing `Order` dataclass (lower surface change than creating `OrderWithId` wrapper).
- **`OrderResponse` deprecation cycle**: 1 release vs. immediate removal. Per §5 — keep 1 release (v0.5.0 removes).
- **Position quantity precision**: `int` (mirror shioaji) vs `Decimal` (honor Binance fractional). Per §4 risks — use `Decimal`, document divergence.
- **EXPIRED status**: collapse into `Cancelled` vs add new `Expired` enum value (would diverge from shioaji). Per §3.5 — collapse into `Cancelled` for shioaji parity. Document in mapping table.
- **Whether to also rename `Order` class fields** (`action="long"` → `action="Buy"` for shioaji vocab parity). Recommend NO — `Order` is a user-input dataclass, breaking it requires touching every caller; lower-value than return-type alignment. Defer to v0.5.0 if appetite.
